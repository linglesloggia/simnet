module documentation

QA tests for the simulator classes.

This scripts provide an easy way to test simulations in an example case comprising:

  • 1 base station;
  • 2 slices;
  • 3 user groups, 2 assigned to slice 1, 1 assigned to slice 2.

The following values can be adjusted as parameters; default values are provided:

  • number of user equipments in each user group;
  • maximum length of packet queues, same for user equipments in each user group;
  • keep or discard transmitted packets, same for user equipmens in each user group;
  • number of packets to generate on each instance, same for user equipmens in each user group;
  • quantity of resources Poor, Fair or Good to generate;
  • an assignment matrix specifies quality and quantity of resources to hand out to slices.

Transport block size, and hence number of packets it can contain, are determined from the following default values:

  • resources categories are Poor, Fair or Good, providing 22, 44, or 88 symbols respectively;
  • channel state determines bits per symbol 0, 1, 2, 4 or 8 in a random sort of equal probability for each size;
  • transport block size is bits per symbol * quantity of symbols;
  • these allow for transport block sizes from 0 to 88*8=704 bits;
  • since size of packets is 40 bits, a transport block may contain 0 to 17 packets.

Function setup_sim creates the simulation scenery. Several test functions show how to verify the simulation scenery and run several types of simulation. This function also shows a way to create other possible sceneries in a relatively simple way, at least easier than defining every entity, which is the way to go for accurate scenery set up.
